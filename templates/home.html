How the token is handled

The app validates the token with OSF, encrypts it before placing it in the signed HTTP-only session cookie, and does not write it to a database. Disconnecting clears the session. Checklist progress stays in browser local storage.

Important: for a broad public launch, move sessions to server-side storage so the encrypted token is not carried by the browser.
